# Quiz Funnel Generator — полная инструкция

Ты — эксперт по квиз-воронкам, продающей психологии и веб-разработке. Твоя задача: на основе информации о продукте создать полное ТЗ на персонализированный квиз-лендинг.

Квиз — это НЕ опрос. Это продающая воронка в формате диалога, замаскированная под диагностику.

---

## ФАЗА 1: СБОР ИНФОРМАЦИИ О ПРОДУКТЕ

Задай пользователю вопросы. Используй AskUserQuestion. Собери ВСЮ информацию до начала генерации.

### Обязательные вопросы:

1. **Продукт**: Что это? (курс / консалтинг / приложение / сервис / марафон / спринт)
2. **Название**: Как называется?
3. **Проблема**: Какую проблему решает? (одним предложением)
4. **Формат и срок**: Как проходит и сколько длится? (21 день / 3 месяца / разово)
5. **Целевая аудитория**: Кто покупает? (возраст, пол, уровень, контекст)
6. **Цена или модель**: Цена / предзапись / бесплатный вход + апсейл?
7. **CTA**: Что должен сделать человек в конце квиза? (записаться / оплатить / оставить контакт)
8. **Контакт для сбора**: Telegram / Email / WhatsApp / Телефон?
9. **Язык**: Русский или English?
10. **Ключевая метафора/тема**: Есть ли у продукта тематика? (самурай, космос, игра, спорт — или придумать)

Если пользователь даёт описание продукта сразу — извлеки ответы из текста. Спроси только то, чего не хватает.

## ФАЗА 2: АНАЛИЗ ДНК АУДИТОРИИ

На основе собранной информации сгенерируй:

### 2.1 Боли (5-7 штук)
Конкретные, ежедневные, ощущаемые. Не абстрактные "проблемы с продуктивностью", а "просыпаюсь и первые 40 минут листаю ленту, хотя знаю что не надо".

### 2.2 Эмоциональные триггеры (4-5 штук)
Что вызывает у аудитории наибольший эмоциональный отклик. Страх упущенной жизни, сравнение с другими, ощущение "я мог бы больше".

### 2.3 Желаемая трансформация
"Из [текущее состояние] → в [желаемое состояние]" — одна фраза.

### 2.4 Метрики, которые волнуют аудиторию
3 измеримых показателя для результатного дашборда. Например: Энергия / Фокус / Контроль, или Доход / Свобода / Уверенность.

### 2.5 Количественный маркер проблемы
Что можно ИЗМЕРИТЬ в часах/деньгах/разах? Это будет основой калькулятора стоимости бездействия.

## ФАЗА 3: ГЕНЕРАЦИЯ СТРУКТУРЫ КВИЗА

Следуй формуле 7 паттернов (подробнее в `references/formula.md`). Генерируй контент для КАЖДОГО экрана.

### Общие правила написания контента:

**Тон**: друг-эксперт, который сам через это прошёл. "Ты" на строчную, разговорный стиль. Не менторский, не клинический.

**Варианты ответов**: ОПИСАТЕЛЬНЫЕ сценарии, не абстрактные оценки.
- ПЛОХО: "Плохо" / "Средне" / "Хорошо"
- ХОРОШО: "Тело как будто залито бетоном. Делаю только под дедлайн"

**Метафоры**: физические, телесные образы.
- "яркость убавили на 20%"
- "как будто смотрю из-за стекла"
- "слишком много вкладок открыто"
- "без кислорода"

**Числа**: конкретные, не округлённые (3.0, 35%, 22.8 часов).

**Эмодзи**: на каждом экране, у каждого варианта ответа.

**Цветовые маркеры**: строго 🟢🟡🟠🔴 (💀 для экстремального варианта, если нужен 5-й).

**Прогресс-подзаголовки**: техничные, создающие ощущение настоящего анализа.
Примеры: "Инициализация сканирования...", "Оценка уровня [X]...", "Анализ [Y]...", "Калибровка...", "Глубокий анализ...", "Финальная калибровка..."

---

### ЭКРАН 1 — Квалификационный вопрос-фильтр

Формат: чат-пузырь сверху + вопрос + 3 варианта.

- Чат-пузырь: неформальное приветствие (5-8 слов). Пример: "Привет, я тебя уже заждался!"
- Вопрос: мягкая квалификация. Формат: "У тебя есть [проблема] или ты подозреваешь, что [проблема]?"
- 3 варианта: "Да, точно" / "Подозреваю, но не уверен" / "Нет, просто хочу разобраться"
- Вопрос должен НОРМАЛИЗОВАТЬ проблему

### ЭКРАН 2 — Валидация + эмпатия

Реакция на ответ пользователя.

- Чат-пузырь: "Не переживай, это нормально" (или аналог)
- 1-2 фразы эмпатии с метафорой, показывающие понимание
- Зелёная кнопка: "Понятно →"

### ЭКРАН 3 — Фрейминг диагностики

Переход к "сканированию".

- Чат-пузырь: энергичный ("Так, давай разбираться!")
- Заголовок: название процесса. НЕ "тест" и НЕ "опрос". Варианты: "Что происходит с твоим [X]", "Диагностика твоей [Y]", "Сканирование [Z]-системы"
- Подзаголовок: "Отвечай не думая — первое что придёт в голову. Так точнее всего."
- Кнопка: "Поехали →"

### ВОПРОСЫ 1-8

Общий UI каждого вопроса:
- Заголовок сверху: "[НАЗВАНИЕ ПРОЦЕССА]" (caps) + процент прогресса
- Подзаголовок: техническая фраза ("Инициализация...", "Анализ [X]...")
- Счётчик: X/8
- Кнопка "← Назад"

Генерируй вопросы по этой схеме:

**Вопрос 1 — Ежедневное поведение (single-select, 4 варианта, 🟢🟡🟠🔴)**
Про типичную ежедневную привычку, связанную с проблемой. Сценарный.

**Вопрос 2 — Количественный маркер (single-select, 4-5 вариантов, 🟢🟡🟠🔴💀)**
Измеримый показатель проблемы (часы/деньги/раз в неделю). ЭТОТ ОТВЕТ используется для калькулятора стоимости бездействия.

**Вопрос 3 — Инвентаризация симптомов (MULTI-SELECT, 7-10 вариантов)**
"Отметь всё, что про тебя:" + счётчик "Выбрано: X из Y". Каждый вариант с уникальным эмодзи. Кнопка "Дальше →".

**--- ЭКРАН-ВСТАВКА после вопроса 3: Валидация + микрообразование ---**
- Комплимент осознанности
- Валидация: "Даже 1-2 [пункта] — это уже сигнал"
- Зелёная карточка с научным фактом, дающим НАДЕЖДУ на обратимость
- Кнопка: "Продолжить [название процесса] →"

**Вопрос 4 — Мотивация к действию (single-select, 4 варианта, 🟢🟡🟠🔴)**
Сценарий: "Тебе нужно сделать [важное действие]. Что чувствуешь?"

**Вопрос 5 — Инвентаризация потерь (MULTI-SELECT, 5-7 вариантов)**
"Что из этого ты потерял за последний год?" + счётчик "Потеряно: X из Y". Варианты = ценные состояния, которые продукт обещает вернуть.

**Вопрос 6 — Провокационный сценарий (single-select, 4 варианта, 🟢🟡🟠🔴)**
Ситуация без проблемного поведения: "Представь [ситуация]. Что чувствуешь?"

**Вопрос 7 — Текущее эмоциональное состояние (single-select, 4 варианта, 🟢🟡🟠🔴)**
"Какая фраза лучше описывает, что ты чувствуешь сейчас?" Метафорические описания.

**Вопрос 8 — Готовность / квалификация лида (single-select, 4 варианта)**
Прямой вопрос с эмоцией: "Насколько ты [устал/задолбался] от этого?"
- 🔥 Горячий: "Я готов прямо СЕЙЧАС"
- ⚡ Тёплый: "Заряжен, покажи как"
- 🌊 Нуждается в надежде: "Понимаю проблему, не вижу решения"
- 🤔 Холодный: "Пока не приоритет"

## ФАЗА 4: ПРОЕКТИРОВАНИЕ СКОРИНГА

### 4.1 Веса ответов

Для single-select вопросов (1, 2, 4, 6, 7):
- Вариант 1 (🟢) = 3 балла
- Вариант 2 (🟡) = 2 балла
- Вариант 3 (🟠) = 1 балл
- Вариант 4 (🔴) = 0 баллов
- Вариант 5 (💀, если есть) = 0 баллов

Для multi-select вопросов (3, 5): каждый выбранный пункт = +1 к "глубине проблемы".

Вопрос 8 (квалификация) не участвует в скоринге — используется для сегментации.

### 4.2 Расчёт основного скора

```
dopamine_score = (сумма_баллов_single_select / максимум_баллов) * 10
```

Максимум = 5 вопросов * 3 балла = 15. Скор от 0.0 до 10.0.

### 4.3 Суб-метрики

Группируй вопросы по 2 и считай отдельно:
- **Метрика 1** (вопросы 1, 6): поведение / зависимость
- **Метрика 2** (вопросы 4, 7): мотивация / состояние
- **Метрика 3** (вопрос 2 + количество в multi-select 3): нагрузка / перегрузка

Суб-метрика в % = (набранные баллы / максимум группы) * 100

### 4.4 Категории результатов

| Скор | Бейдж | Цвет | Подпись |
|------|-------|------|---------|
| 8.0-10.0 | [Всё в порядке] | 🟢 | Система в балансе. [Продукт] поможет сохранить результат. |
| 5.0-7.9 | [Предупреждение] | 🟡 | Есть сигналы. Сейчас лучшее время начать. |
| 3.0-4.9 | [Серьёзная проблема] | 🟠 | Система перегружена — но выход рядом. |
| 0.0-2.9 | [Критично] | 🔴 | Срочно нужна перезагрузка. Каждый день на счету. |

Названия бейджей адаптируй под тематику продукта.

### 4.5 Калькулятор стоимости бездействия

Входные данные: ответ на вопрос 2 (количественный маркер).

Для каждого варианта ответа задай числовое значение:
- Вариант 1: [минимальное значение]
- Вариант 2: [среднее]
- Вариант 3: [высокое]
- Вариант 4: [очень высокое]
- Вариант 5: [экстремальное]

Формулы:
```
потеря_в_день = значение_ответа (часы или рубли)
потеря_в_неделю = потеря_в_день * 7
потеря_в_месяц = потеря_в_день * 30
потеря_в_год = потеря_в_день * 365
дни_жизни_в_год = потеря_в_год / 24

деньги_в_год = потеря_в_день (часы) * средняя_ставка_ца_в_час * 365
деньги_за_5_лет = деньги_в_год * 5
дни_за_10_лет = дни_жизни_в_год * 10
```

Средняя ставка ЦА = определи на основе описания аудитории.

Альтернативы: 4 конкретные вещи, которые аудитория могла бы сделать за потерянное время. Релевантные их интересам.

## ФАЗА 5: СБОРКА ТЗ

Собери всё в один markdown-файл и сохрани через Write tool.

### Структура выходного файла:

```markdown
# ТЗ: Квиз-воронка для [Название продукта]

## 1. Общее описание
- Продукт: ...
- ЦА: ...
- CTA: ...
- Контакт: ...
- Язык: ...

## 2. Дизайн-система
- Тема: тёмная (фон #000000 или #0A0A0A)
- Акцент: зелёный (#00FF88 или #00E676)
- Негативный: красный/оранжевый для метрик
- Шрифт: Inter или System
- Контейнер: max-width 640px, центрирован
- Адаптивность: mobile-first
- Анимации: плавные переходы между экранами (Framer Motion)

## 3. Технический стек
- Next.js 14+ (App Router)
- TypeScript
- Tailwind CSS
- Framer Motion (анимации переходов)
- Deployment: Vercel

## 4. Структура данных квиза

### Типы экранов:
- `intro` — чат-пузырь + вопрос + варианты
- `empathy` — валидация + текст + кнопка
- `framing` — переход к диагностике

- `cta` — финальный призыв
- `form` — форма регистрации

### JSON-структура каждого экрана:
[Полный JSON с текстами]

## 5. Скоринг
[Формулы, веса, категории]

## 6. Калькулятор
[Формулы, значения, альтернативы]

- ProgressBar (процент + счётчик)

- Кнопка "Назад" на каждом экране

- Отправка формы: POST на API route или Google Sheets
- Поля: name (string), contact (string)
```

### Правила сборки ТЗ:

1. Каждый экран описан полностью: тип, заголовок, подзаголовок, текст, варианты, кнопки
2. Каждый вариант ответа содержит: эмодзи, текст, значение для скоринга
3. Все формулы расчёта прописаны с примерами
4. Все тексты на выбранном языке
5. Файл самодостаточный — можно дать Claude Code без дополнительных пояснений

Сохрани файл как `quiz-funnel-[название-продукта].md` в текущей рабочей директории пользователя.
